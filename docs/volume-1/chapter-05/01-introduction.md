# Strings, Numbers, and Dates

## Introduction

Strings, Numbers, and Dates matters because text, numeric data, BigInt, regular expressions, timestamps, and time calculations. A professional JavaScript engineer is expected to reason beyond syntax: what the engine creates, what the host environment provides, which values live in memory, and which choices produce maintainable systems.

The recurring misconception is this: A string character is not always one UTF-16 code unit, money is not safely represented by arbitrary floating-point arithmetic, and a date without a timezone policy is a future incident report.

In interviews, this topic is a signal. Interviewers are rarely testing whether you can recite trivia. They are testing whether you can predict behavior, communicate trade-offs, and write code that holds up when requirements become less friendly.

## Learning Objectives

By the end of this chapter, you will be able to:

- Explain String methods and template literals.
- Explain Unicode and user-visible text.
- Explain Regular expression basics.
- Explain Number and Math objects.
- Explain BigInt for integer precision.
- Explain Floating-point precision.
- Explain Date object, timestamps, formatting, and calculations.
- Draw a memory or execution diagram for the topic.
- Explain browser and Node.js differences where they matter.
- Answer beginner, intermediate, senior, and FAANG-style interview questions confidently.
