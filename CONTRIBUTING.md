# Contributing

Thank you for helping build `JavaScript Mastery for FAANG Interviews`.

This project accepts only complete, reviewable book material. A chapter is complete when a reader can study it without relying on missing future sections.

## Contribution Standards

- Explain the concept, the reason it exists, the problem it solves, and how it behaves internally.
- Include runnable JavaScript examples in `code/`.
- Include Mermaid diagrams in `diagrams/` when the topic has flow, structure, memory, or scheduling behavior.
- Include interview explanations and follow-up questions.
- Include exercises, solutions, complexity analysis, MCQs, and revision notes.
- Keep examples realistic and production-oriented.
- Use cross references instead of repeating long explanations.

## Style

- Prefer clear, direct language.
- Teach reasoning before syntax.
- Avoid motivational filler.
- Do not add placeholder files.
- Do not merge unfinished chapters.

## Pull Request Checklist

- [ ] Markdown renders without broken links.
- [ ] Code examples run independently.
- [ ] Mermaid diagrams render.
- [ ] Chapter follows `docs/book-production-standard.md`.
- [ ] Interview questions include follow-ups.
- [ ] Exercises include hints, solutions, complexity, and alternatives.
